## 五、问题记录与解决方法

### 1. 路径`\1` `\2`被解释为八进制转义
**问题**：`'D:\Semems\1AI\DX0001\12.png'` 中 `\1` `\2` 被Python解释为SOH/STX。
**解决**：用 `chr(92).join(['D:', 'Semems', '1AI', ...])` 构建路径。

### 2. 美图未关闭时重复点击AI工具/AI贴图关闭了添加按钮
**问题**：美图还在运行就重复点AI工具→AI贴图，新增图片按钮被关闭。
**解决**：美图未关闭时直接点新增图片，不点AI工具和AI贴图。

### 3. 尾数1从右下角拖动导致图片缩放
**问题**：从右下角拖动了尾数1的贴图，导致拉伸变小。
**解决**：尾数1必须从**贴图内部**(2100,720)开始拖，加Alpha偏移。

### 4. 首次打开胚衣图片大小不对
**问题**：第一次打开美图时胚衣显示大小异常。
**解决**：关图→不保存→重新打开胚衣，图片大小就正常了。

### 5. 黑背尾数1误加了旋转
**问题**：黑背尾1也和白背一样做了旋转，位置异常。
**解决**：黑背尾1**不旋转**，直接选中→拖动→混合→保存。

### 6. 剪贴板用subprocess clip不生效
**问题**：`subprocess.run(['clip'], input=path.encode())` 无法正确设置剪贴板。
**解决**：用 `powershell -Command Set-Clipboard -Value "路径"`。
路径用 `chr(92).join(parts)` 构建。

### 7. 保存同名文件时覆盖确认
**问题**：同一文件夹先尾2后尾1保存时同名文件冲突，美图弹出覆盖确认弹窗。
**解决**：保存后如有覆盖提示，用 **Alt+Y** 确认覆盖，或点击覆盖确认按钮(2077,787)。
**注意**：覆盖弹窗是Qt自定义窗口，`EnumWindows` 用 `'ToolSa' in cls.value` 检测，
回调中**必须调用** `GetClassNameW(h, cls, 64)` 获取类名后再判断。

### 8. 美图保存文件名根据胚衣自动变化
**问题**：白T保存为`白正2_副本.jpg`，黑T保存为`黑正2_副本.jpg`/`黑背2_副本.jpg`。
脚本硬编码检测`白正2_副本.jpg`导致黑T保存验证失败。
**解决**：`wb_sticker_tail2.py`中：白T→`白正2_副本.jpg`，黑T→`黑正2_副本.jpg`。
`wb_sticker_tail1.py`中：白T→`白正2_副本.jpg`，黑T→`黑背2_副本.jpg`。
**教训**：保存验证必须根据`is_black`参数选择对应文件名。

### 9. 保存路径最终方案：pyperclip + 双重验证（v2.2关键修复）
**问题根因**：保存弹窗是Qt DirectUI控件
**解决方案（v2.2）**：
1. **pyperclip** 替代 PowerShell Set-Clipboard
2. **双重验证**：粘贴前读取确认点击中了→粘贴后读取确认改成功了
3. **Ctrl+V分步慢速**（0.05/0.03/0.03/0.1秒）
4. **坐标(2200,555)**（爸爸实测确认）
5. **先删旧文件**避免覆盖弹窗干扰

### 10. patch工具破坏代码结构（v2.3修复）
**问题**：使用patch工具替换click函数时，代码被改成了嵌套定义：
```python
def click(x, y, d=0.05):
    def click(x, y, d=0.05):  # ← 嵌套定义！
        u.SetCursorPos(...)
```
导致click函数什么都不做，所有点击失效——AI工具、新增图片、路径框全点不了。

**解决**：每次patch后必须**编译检查** (`python3 -m py_compile`) 和**功能测试**。

## 九、v2.3.0 新增经验（2026-06-21）

### 1. 保存路径必须双重验证

**问题**：简化代码时去掉了路径框点击和剪贴板回读验证，导致 Ctrl+V 粘贴到错误位置，文件存到 DX0022。

**正确做法**：
```
click 路径框(2200,555)
Ctrl+A → Delete → pyperclip回读验证含'DX'（确认点中路径框）
清空 → 粘贴新路径 → Ctrl+V → Ctrl+A → Ctrl+C回读验证
```

### 2. tail1 输出文件名

tail1 处理背面，输出名应为 `白背2_副本.jpg` 而非 `白正2_副本.jpg`。

**⚠️ 代码中易混淆点**：
- tail1：输出 `白背2_副本.jpg` / `黑背2_副本.jpg`
- tail2：输出 `白正2_副本.jpg` / `黑正2_副本.jpg`

### 3. 保存后验证循环

不要用 60次×0.5s（30秒），改为 25次×0.2s（5秒），文件出现即返回。

### 4. import_file 用 pyperclip 替代 SendMessageW

文件打开弹窗设路径改用 pyperclip + Ctrl+V + Enter，比 WM_SETTEXT 可靠。

### 5. 返回值检查

背（tail1）失败时不应继续正（tail2），否则残留弹窗会导致操作错乱。
主入口已加返回值检查：`if not run_tail1_fn(...): 跳过 run_tail2_fn`

### 6. 点击路径框后需等 0.3s

点击路径框后立即按 Ctrl+A 可能来不及聚焦，需等 0.3s 再操作。

### 7. 保存对话框查找

`key_comb(0x11, 0x53)`（Ctrl+S）后等 1.5s 内找 `ToolSa` 类窗口。改短为 15次×0.1s，弹窗通常瞬间出现。<｜end▁of▁thinking｜>
