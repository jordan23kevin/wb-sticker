## 3. 核心文件职责

### 3.1 `wb_sticker.py` — 主入口

负责扫描 DX 文件夹，根据文件名判断类型，调度对应脚本。

**用法：**
```bash
uv run --with pillow --with numpy --with pyperclip --with pywin32 python wb_sticker.py              # 白T
uv run --with pillow --with numpy --with pyperclip --with pywin32 python wb_sticker.py --black       # 黑T
uv run --with pillow --with numpy --with pyperclip --with pywin32 python wb_sticker.py DX0001        # 指定文件夹
```

### 3.2 `wb_sticker_tail2.py` — 正面（尾数2）

给胚衣贴正面图案：
- 白T胚衣：`白正2.jpg`
- 黑T胚衣：`黑正2.jpg`
- 操作：右下角拖动 → 旋转6像素 → 混合模式

### 3.3 `wb_sticker_tail1.py` — 背面（尾数1）

给胚衣贴背面图案：
- 白T胚衣：`白背2.jpg`
- 黑T胚衣：`黑背2.jpg`
- 操作：Alpha加权定位 → 内部拖动 → **黑背不旋转** → 混合模式

### 3.4 `03-美图贴图/` — 原始参考

美图贴图原项目的完整脚本和文档。已在 Lovart 项目跑通 **528/528 零失败**。坐标使用 **窗口相对坐标**，与 WB 版的**绝对坐标**不同。

## 4. 命名规则

| 文件名含（大小写不敏感） | 类型 |
|:------------------------:|:----:|
| **BW** 或 **WB** | **先背后正**各贴一次 |
| **B**（不含BW/WB） | **背**（白背2.jpg / 黑背2.jpg） |
| **W**（不含BW/WB） | **正**（白正2.jpg / 黑正2.jpg） |
| 无B/W，最后一位数字1 | 背（兼容旧命名） |
| 无B/W，最后一位数字2 | 正（兼容旧命名） |

判断顺序：BW/WB优先 → B → W → 数字尾数。其余跳过。

```
    │       输出: DX0020\白正2_副本.jpg / 黑正2_副本.jpg
    │
    └── BW/WB → 先尾数1（背），再尾数2（正）
```

## 6. 白T vs 黑T 差异

| 项目 | 白T | 黑T |
|:----|:---:|:---:|
| 胚衣 | 白正2.jpg / 白背2.jpg | 黑正2.jpg / 黑背2.jpg |
| 尾数2拖动 | 右下角→(2166,705) | ✅ 相同 |
| 尾数2旋转 | ✅ 需要 | ✅ 需要 |
| 尾数2混合 | 三步 | **两步** |
| 尾数1拖动 | 内部+Alpha加权(TARGET_X=2115) | 内部+Alpha加权(TARGET_X=**2110**) |
| 尾数1旋转 | ✅ 需要 | **❌ 不需要** |
| 尾数1混合 | 三步 | **两步** |
