## 3. 核心文件职责

### 3.1 `wb_sticker.py` — 主入口 v2.3.0

负责扫描 DX 文件夹，根据文件名判断类型，调度对应脚本。
- 白T/黑T 双模式（`--black` 参数）
- 支持指定文件夹或全部扫描
- **返回值检查**：背失败时自动跳过正，避免残留弹窗干扰

### 3.2 `wb_sticker_tail2.py` — 正面（尾数2）v2.3.0

给胚衣贴正面图案：
- 白T胚衣：`白正2.jpg`，输出 `白正2_副本.jpg`
- 黑T胚衣：`黑正2.jpg`，输出 `黑正2_副本.jpg`
- 操作：右下角拖动 → 旋转 → 混合模式

### 3.3 `wb_sticker_tail1.py` — 背面（尾数1）v2.3.0

给胚衣贴背面图案：
- 白T胚衣：`白背2.jpg`，输出 `白背2_副本.jpg`
- 黑T胚衣：`黑背2.jpg`，输出 `黑背2_副本.jpg`
- 操作：Alpha加权定位 → 内部拖动 → **黑背不旋转** → 混合模式

### 3.4 `03-美图贴图/` — 原始参考

美图贴图原项目的完整脚本和文档，528/528 零失败。使用窗口相对坐标。

## 6. 保存流程（关键修复）

### 保存路径设置（轻量验证）
```
ff(保存弹窗)
  → import pyperclip
  → 点击路径框2次(2200,555)       # 确保聚焦
  → Ctrl+A → Ctrl+C → 读取剪贴板  # 轻量验证是否点中路径框
  → 显示当前路径（如 [path] DX0021）
  → Delete → 粘贴新路径            # 设置正确路径
  → Ctrl+V
  → 设质量100 → 点击保存按钮
  → 验证循环 15次×0.2s             # 3s快速检测文件出现
```

### 关键时间优化

| 步骤 | 原耗时 | 优化后 |
|:----|:-----:|:-----:|
| 等保存弹窗 | 1.5s | **0.5s** |
| 路径设置+验证 | ~2s | **~0.5s** |
| 等文件出现 | 5s | **3s**（通常瞬间） |
| 拖动步数 | 12步×5ms | **8步×3ms** |
| 旋转步数 | 3步×5ms | **2步×3ms** |

## 7. 修复记录

### 问题1：保存路径错误（2026-06-21）
**症状**：处理 DX0020 但文件保存到 DX0022
**根因**：简化代码时去掉了路径框的点击和双重验证，Ctrl+V 没粘贴到正确位置
**修复**：恢复原版保存路径双重验证逻辑

### 问题2：tail1 输出文件名错误（2026-06-21）
**症状**：白背处理后验证找不到文件，超时报错
**根因**：tail1 输出名写成了 `白正2_副本.jpg`（应该是 `白背2_副本.jpg`）
**修复**：`白正2` → `白背2`

### 问题3：保存验证循环重复（2026-06-21）
**症状**：恢复原版代码后有两个验证循环（慢30s + 快5s），慢的消耗完才用快的
**修复**：删除慢循环（60次×0.5s），保留快循环（25次×0.2s）

### 问题4：import_file 使用 SendMessageW 设路径失败
**症状**：文件打开弹窗无法自动填入路径
**修复**：改用 pyperclip + Ctrl+V 粘贴路径 + Enter

### 问题5：保存路径轻量验证（v2.4.0）
**症状**：不加验证时路径框没点中，路径设不进去
**修复**：点击2次路径框 + Ctrl+A/Ctrl+C 读回确认聚焦，然后粘贴新路径
**耗时**：仅增加 ~0.3s，但大幅提高可靠性
